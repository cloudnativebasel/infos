# Cloud Native terminology

Here are definitions for each item in the alphabetical list of Cloud Native terms:

- **Auto Scaling:** Capability to automatically adjust the number of resources based on workload demands.

- **Abstraction**: In computing, abstraction is a process by which unnecessary details are hidden from the user. The user can utilize the functionality without knowing how it works internally.
- 
- **Agile Software Development**: A set of principles for software development under which requirements and solutions evolve through the collaborative effort of self-organizing cross-functional teams.

- **API Gateway**: A server that acts as an API front-end, receives API requests, enforces throttling and security policies, passes requests to the back-end service and then passes the response back to the requester.

- **Application Programming Interface (API)**: A set of rules and protocols for building and interacting with software applications.

- **Autoscaling**: The process by which a cloud service automatically adjusts the amount of computational resources based on the traffic load.

- **Bare Metal Machine**: Physical servers dedicated to a single tenant, meaning one tenant has complete control, flexibility, and security over their server.

- **Blue-Green Deployment:** Technique where two identical environments (blue and green) are used for deployment to minimize downtime during updates.

- **Canary Deployment**: A strategy for rolling out releases to a subset of users or servers. The idea is to first deploy the change to a small subset of servers, test it, and then roll the change out to the rest of the servers.

- **Chaos Engineering**: The discipline of experimenting on a system in order to build confidence in the systemâ€™s capability to withstand turbulent conditions in production.

- **CI/CD (Continuous Integration/Continuous Delivery):** Practices and tools for automating the build, test, and deployment of applications.

- **Client-Server Architecture**: A network design where client devices such as desktops, laptops, and mobile devices use a network to access resources such as files and applications stored on a server.

- **Cloud Computing**: The on-demand delivery of IT resources over the Internet with pay-as-you-go pricing.

- **Cloud Native Apps**: Applications that are designed to take full advantage of cloud computing frameworks, composed of microservices, packaged in containers, and managed on elastic infrastructure through agile DevOps processes and continuous delivery workflows.

- **Cloud Native:** Refers to an approach to software development and deployment that leverages cloud computing technologies and principles.

- **Cloud Native Databases:** Databases designed and optimized for deployment and management in cloud-native environments.

- **Cloud Native Governance:** Practices and policies to ensure compliance, security, and accountability in cloud-native environments.

- **Cloud Native Networking:** Networking architecture and tools designed to support cloud-native environments, enabling scalability and resiliency.

- **Cloud Native Security:** Set of practices and tools to ensure the security and compliance of cloud-native applications and infrastructure.

- **Cloud-Native Infrastructure:** Infrastructure components, such as networking, storage, and compute resources, designed for cloud-native environments.

- **Cloud Native Technology**: Technology that is designed specifically for a cloud-based environment rather than a traditional data center environment.

- **Cloud-Native Testing:** Approaches and tools for testing cloud-native applications, including container-based testing and chaos engineering.

- **CNCF (Cloud Native Computing Foundation):** A vendor-neutral organization that hosts and promotes open-source cloud-native projects.

- **Cluster**: In computing, a cluster is a group of servers and other resources that act like a single system and enable high availability and, in some cases, load balancing and parallel processing.

- **Containers:** Lightweight, isolated runtime environments that package applications and their dependencies for consistent and efficient deployment.

- **Container Orchestration**: The automated arrangement, coordination, and management of computer systems, middleware, and services. It helps in managing the lifecycle of containers in large dynamic environments.

- **Continuous Delivery (CD)**: A software development practice where code changes are automatically built, tested, and prepared for a release to production.

- **Continuous Deployment (CD)**: A software development practice in which every code change goes through the entire pipeline and is put into production automatically, resulting in many production deployments every day.

- **Continuous integration (CI)**: A software development practice where developers regularly merge their code changes into a central repository, after which automated builds and tests are run.

- **Datacenter**: A facility used to house computer systems and related components, such as telecommunications and storage systems.

- **DevOps:** Cultural and technical approach that combines development and operations teams to automate software delivery processes.

- **DevSecOps**: A philosophy or culture promoting the inclusion of security practices within DevOps processes.

- **Distributed Apps**: Applications where parts of them run concurrently on multiple machines.

- **Distributed System**: A system whose components are located on different networked computers, which communicate and coordinate their actions by passing messages to each other.

- **Edge Computing**: A method of optimizing cloud computing systems by performing data processing at the edge of the network, near the source of the data.

- **Event Streaming**: A practice of sending data from one application to others in real-time over the internet. This is often done through the use of publish-subscribe systems.

- **Elasticity:** Ability to automatically scale resources up or down based on workload demands.

- **Event-Driven Architecture:** Approach where applications respond to and emit events as the primary means of communication and coordination.

- **Function-as-a-Service (FaaS):** A serverless compute offering that enables running individual functions in the cloud.

- **Horizontal Scaling**: A method of adding more machines to a network to improve performance and space for data storage rather than replacing existing machines with more powerful machines (vertical scaling).

- **Hybrid Cloud:** Environment that combines public and private clouds, allowing data and applications to be shared between them.

- **Idempotence**: The property of certain operations in mathematics and computer science whereby they can be applied multiple times without changing the result beyond the initial application.

- **Immutable Infrastructure:** Infrastructure that is treated as disposable and replaced rather than updated or modified.

- **Infrastructure-as-a-Service (IaaS)**: Online services that provide high-level APIs used to dereference various low-level details of underlying network infrastructure like physical computing resources, location, data partitioning, scaling, security, backup etc.

- **Infrastructure-as-Code (IaC):** Practice of managing infrastructure resources through declarative configuration files.

- **Kubernetes:** An open-source container orchestration platform that automates deployment, scaling, and management of containerized applications.

- **Loosely Coupled Architecture**: An approach to designing systems with simple, singular purpose components that can work together.

- **Microservices:** Architectural style where applications are composed of small, loosely coupled services that can be independently developed, deployed, and scaled.

- **Microservices Architecture**: An architectural style that structures an application as a collection of small autonomous services, modeled around a business domain.

- **Monolithic Apps**: A software application which is designed as a single and indivisible unit. The monolithic architectural style is a unified model for the design of a software program.

- **Multitenancy**: A reference to the mode of operation of software where multiple independent instances of one or multiple applications operate in a shared environment. The instances (tenants) are logically isolated, but physically integrated.

- **Mutual Transport Layer Security (mTLS)**: A security protocol in which each party in a communication session certifies the other party's identity. It's called mutual because both the client and server certify each other's identities.

- **Nodes**: In computing, a node is a device that can send, receive, or forward information over a network. In the context of a Kubernetes cluster, a Node is a worker machine and may be a VM or physical machine.

- **Observability:** Ability to gain insight into the behavior and performance of distributed systems through monitoring, logging, and tracing.

- **Orchestration:** Automated coordination and management of multiple components, such as containers, in a cloud environment.

- **Policy as Code (PaC)**: An approach to writing code that automates the implementation, compliance, and remediation of policies.

- **Polyglot Persistence:** Using different types of databases or data storage technologies within an application to fit specific data needs.

- **Portability**: The usability of the same software in different environments.

- **Reliability**: The ability of a system or component to perform its required functions under stated conditions for a specified period of time.
 
- **Role-Based Access Control (RBAC)**: A policy-neutral access-control mechanism defined around roles and privileges.

- **Scalability:** Ability of a system to handle increased workload by adding resources without affecting performance.

- **Self Healing**: The process by which a system detects and corrects faults that may occur.

- **Security Chaos Engineering**: The discipline of asserting system security in the presence of turbulent conditions.

- **Service Discovery:** Mechanism for locating and connecting to available services in a dynamic, distributed system.

- **Service Mesh:** Infrastructure layer that provides advanced networking, observability, and security features for microservices communication.

- **Sidecar Pattern:** Design pattern where a separate component runs alongside a primary service to provide additional functionality.

- **Serverless**: A cloud computing execution model in which the cloud provider runs the server, and dynamically manages the allocation of machine resources, allowing developers to focus on writing and deploying code.

- **Service**: A component that provides a specific functionality. In the context of microservices, a Service is a component that can be deployed independently and communicate over a network.

- **Service Discovery**: In microservices, service discovery is a way for services to find the network location of other services.

- **Service Mesh**: A dedicated infrastructure layer built right into an app. This visible infrastructure layer can document how well (or not) different parts of an app interact, so developers can optimize communication and avoid downtime.

- **Service Proxy**: An intermediary that controls and coordinates the communication among microservices.

- **Shift Left**: A practice intended to find and prevent defects early in the software delivery process.

- **Site Reliability Engineering**: A discipline that incorporates aspects of software engineering and applies them to infrastructure and operations problems.

- **Stateful Apps**: An application that saves client data from the activities of one session for use in the next session. The data that is saved is often called the application's state.

- **Tightly Coupled Architectures**: An architecture where each component and its associated components must be present in order for code to be executed or compiled.

- **Transport Layer Security (TLS)**: A cryptographic protocol designed to provide communications security over a computer network.

- **Vertical Scaling**: The process of adding resources such as processing power and memory to a single node in a system.

- **Virtual Machine**: An emulation of a computer system that provides the functionality of a physical computer.

- **Virtualization**: The process of creating a virtual version of something, including but not limited to a virtual hardware platform, storage device, network resources, and operating systems.

- **Zero Trust Security:** A security model centered on the belief that organizations should not automatically trust anything inside or outside its perimeters and instead must verify anything and everything trying to connect to its systems before granting access.
